Same DiagramEditor as the default demo, but with a custom ThemeRegistry theme applied to the host before mount. No CSS overrides — only CSS variable tokens set on the container element.

Theme via ThemeRegistry

Call ThemeRegistry.apply(host, 'my-theme') before or after mounting the editor. All components inside inherit the CSS variables automatically.

Partial Themes

Themes only need to specify the tokens they change. Every unset token falls back to the built-in default, so a theme that only tweaks accent color is one line.

Custom Layouts Too

Apply the theme to any wrapper div, not just a DiagramEditor host. Hand-wired standalone controls inside that div will also pick up the theme automatically.

Custom Toolbar Layout

This demo removes the first toolbar section (the file actions and their separator) by passing a custom layout into DiagramEditor. We start from DEFAULT_LAYOUT, drop everything through the first |, and keep the rest.
